# Документация к курсовому проекту: Система мониторинга нарушений дисциплины

## 1. Описание проекта
Приложение представляет собой информационную систему для учебного заведения, позволяющую подавать обращения (в том числе анонимные) о нарушениях дисциплины (буллинг, списывание, прогулы и т.д.). Система предназначена для трех ролей: Ученик, Преподаватель и Администратор.

## 2. Архитектура и технологии
Проект построен по архитектурному паттерну **MVVM** (Model-View-ViewModel) с использованием следующих технологий:
- **UI Фреймворк:** Avalonia UI (XAML)
- **Язык программирования:** C# (шаблон .NET 8)
- **База данных:** PostgreSQL
- **ORM:** Entity Framework Core
- **Дополнительные пакеты:** CommunityToolkit.Mvvm, BCrypt.Net-Next (для хэширования паролей)

## 3. Структура Базы Данных (Модели)
Ранее в проекте использовались перечисления (Enums) для ролей и статусов, однако они были полностью заменены на полноценные модели (сущности базы данных) для обеспечения гибкости.

В системе используются следующие таблицы:
1. **Users (Пользователи)**
   - `Id` (int) - Уникальный идентификатор
   - `FullName` (string) - ФИО
   - `Login` (string) - Логин для входа
   - `PasswordHash` (string) - Хэшированный пароль
   - `RoleId` (int) - Внешний ключ на таблицу Roles
   - `CreatedAt` (DateTime) - Дата регистрации
2. **Roles (Роли)**
   - `Id` (int)
   - `Name` (string) - Название роли (Student, Teacher, Admin)
3. **Categories (Категории нарушений)**
   - `Id` (int)
   - `Name` (string) - Название (Списывание, Буллинг и т.д.)
   - `SeverityLevel` (int) - Уровень серьезности
4. **Reports (Обращения)**
   - `Id` (int)
   - `AuthorId` (int) - Внешний ключ на автора (User)
   - `CategoryId` (int) - Внешний ключ на категорию (Category)
   - `Description` (string) - Описание ситуации
   - `StatusId` (int) - Внешний ключ на статус (ReportStatus)
   - `IsAnonymous` (bool) - Флаг анонимности
   - `CreatedAt` (DateTime) - Дата создания
5. **ReportStatuses (Статусы обращений)**
   - `Id` (int)
   - `Name` (string) - Название статуса (New, InProgress, Resolved, Rejected)

> **ВАЖНОЕ ПРИМЕЧАНИЕ ПО БД:**
> Так как `Role` и `ReportStatus` теперь являются полноценными таблицами (а не Enum), перед полноценным использованием системы администратору или разработчику необходимо заполнить эти таблицы базовыми значениями в СУБД (Student, Teacher, Admin для ролей и New, InProgress, Resolved, Rejected для статусов).

## 4. Структура проекта
- **Models/** — Классы сущностей для EF Core.
- **Data/** — `ApplicationDbContext.cs` для настройки связи с PostgreSQL и конфигурации таблиц.
- **ViewModels/** — Логика представления:
  - `LoginViewModel` / `RegisterViewModel` — Авторизация и регистрация.
  - `MainWindowViewModel` — Управление навигацией в зависимости от роли пользователя.
  - `DashboardPageViewModel` — Главная панель со статистикой обращений.
  - `CreateReportViewModel` — Логика подачи жалобы.
  - `MyReportsViewModel` — История жалоб текущего пользователя.
  - `AdminReportsViewModel` / `AdminUsersViewModel` — Панели управления для администратора.
- **Views/** — Окна и страницы (XAML-файлы):
  - `MainWindow.axaml` — Главный контейнер с боковым меню.
  - `LoginWindow.axaml` / `RegisterWindow.axaml` — Окна входа (с упрощенным, аккуратным интерфейсом).
  - Папки со страницами приложения (`DashboardPage`, `AdminReportsPage` и др.).

## 5. Инструкция по запуску
1. Убедитесь, что в `ApplicationDbContext.cs` указана актуальная строка подключения к базе данных PostgreSQL.
2. Создайте миграцию для применения изменений схемы (замена Enum на таблицы):
   ```bash
   dotnet ef migrations add ReplaceEnumsWithModels
   ```
3. Обновите базу данных:
   ```bash
   dotnet ef database update
   ```
4. Внесите базовые записи в таблицы `Roles` и `ReportStatuses` (вручную через pgAdmin или скриптом).
5. Запустите проект командой:
   ```bash
   dotnet run
   ```
